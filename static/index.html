Serulian is a modern language and toolkit for developing scalable web and mobile applications
Install Serulian
  • Strong but flexible typing

    A strong, extensible and generic type system with classes, implicit interfaces and native immutable structures
  • Safe, strong and standard interop with JS

    Builds on existing web standards to weave with plain JavaScript, without resorting to voodoo magic or broken types
  • Simple and powerful reuse of packages

    Opinionated but extensible packaging and import system that solves dependency hell
  • Straightforward and readable asynchronous code

    Readable asynchronous code without callback hell or superflouous (and verbose) keywords
  • Inline declarative markup

    Extensible and generic declarative markup for easy definition of structures such as DOM

Fully verified code that won't break at runtime

from "github.com/Serulian/debuglib" import Log class MyClass { function<string> GetValue() { return 'hello world!' } } class AnotherClass { function<int> GetValue() { return 42 } } interface Logger<T> { function<T> GetValue() } function<void> Run() { // Full generics var<Logger<string>> l1 = MyClass.new() var<Logger<int>> l2 = AnotherClass.new() Log(l1.GetValue()) Log(l2.GetValue()) }

Type mismatches or missing members are reported as errors

from "github.com/Serulian/debuglib" import Log class MyClass { function<string> GetValue() { return 'hello world!' } } function<void> Run() { Log(MyClass.new().getValue()) // Will fail to compile with an error. }

Properly versioned imports prevents code breakage

// Import HEAD of default branch from "github.com/Serulian/debuglib" import Log // Import HEAD of a specific branch from "github.com/Serulian/debuglib:master" import MasterLog // Import a specific commit from "github.com/Serulian/debuglib:ce6e94b" import SpecificLog // Import a specific version/tag from "github.com/Serulian/debuglib@v0.0.1" import VersionedLog

Tooling for seamlessly working with versioned packages

# Freeze to the latest commit
serulian imports freeze myfile.seru github.com/somenamespace/somerepo

# Unfreeze back to HEAD
serulian imports unfreeze myfile.seru github.com/somenamespace/somerepo

# Update the minor version of semantic versioned package
serulian imports update myfile.seru github.com/somenamespace/somerepo

# Upgrade the major version of semantic versioned package
serulian imports upgrade myfile.seru github.com/somenamespace/somerepo

Working with native JavaScript is as simple as importing an interface

// Import WebIDL like any other import. from webidl`github.com/Serulian/debuglib:master` import console from webidl`github.com/Serulian/corelib:master` import JSON function<void> Run() { // Call the native JS type with full type checking. console.log(&'hello world') // Note the & to send the native string (which String wraps) to // the calls. console.log(JSON.stringify(JSON.parse(&'{"hello": "world"}'))) }

Asynchronous code without callbacks, then or async

from "github.com/Serulian/debuglib" import Log from "github.com/Serulian/request" import Get function<void> Run() { // Get via XHR. This is an *async* call, but is written in sync style. // Log will only be called once the call has succeeded. result := Get('helloworld.txt') Log(result.Text) result2 := Get('hiuniverse.txt') Log(result2.Text) }

Declare components in HTML-like syntax, without being tied to a specific framework or library

from webidl`github.com/Serulian/virtualdom` import document from "github.com/Serulian/component" import RenderComponent from "github.com/Serulian/virtualdom" import Context, Div struct myComponentProps { message string } class MyComponent { var<string> message constructor Declare(props myComponentProps) { return MyComponent{ message: props.message, } } function<any> Render(context Context) { return <Div style="background-color: lightblue; padding: 6px;"> the message: {this.message} </Div> } } function<void> Run() { RenderComponent(<MyComponent message="the message!" />, document.body) }
